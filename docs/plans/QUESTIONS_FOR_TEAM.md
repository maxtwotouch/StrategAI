## 1. Hardware & Performance Benchmarks

### Q1.1 — GPU Used for Asset Generation Benchmarks
The report states "1.2–4 seconds per image" and references an **RTX 5090**. This GPU does not exist in the real world (as of mid-2026, the RTX 4090 is the flagship with 24 GB VRAM).

- **What GPU was actually used for benchmarking?** (exact model and VRAM)
- **Were the 1.2–4s timings measured or estimated?** If measured, do you have logs or screenshots?
- **The report claims "consumer GPUs (8–12 GB VRAM)"** — does this refer to the benchmarking hardware or the target deployment hardware?

### Q1.2 — Cache Hit Rate
The report claims "~80% cache hit rate."

- **How was this measured?** What workload (how many turns, how many civs, what map size)?
- **Is this a measured number or an estimate?** If measured, where are the logs?

The report claims "<100ms API response."

- **Was this load-tested?** With what tool (e.g., `ab`, `wrk`, `locust`)?

The report claims "2–4s LLM decision time."

- **Is this wall-clock time from request to response?** Does it include tool-call round-trips?
- **What GPT-4 model snapshot was used?** (e.g., `gpt-4-0613`, `gpt-4-turbo-2024-04-09`)

The report claims "60 FPS map rendering."
