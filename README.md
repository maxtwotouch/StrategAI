# TopDownMedievalPixelArt Dataset Pipeline

Project for dataset generation, curation, and final export for fine-tuning.

## What This Project Produces

The pipeline has two dataset stages:

1. **Raw dataset (`raw/`)**
   - Generated images with embedded PNG metadata (prompt row + image path).
   - Used for visual review and curation (delete bad PNGs).

2. **HF dataset (`hf/`)**
   - Clean export for Hugging Face `imagefolder` publishing and fine-tuning.
   - Produced only from assets whose PNGs still exist after curation.
   - Side-by-side `.txt` caption files + `metadata.jsonl`.

## Asset Types

| Type | Description | Default Ratio |
|---|---|---|
| `structure` | Buildings (cottages, towers, mills, keeps, etc.) | 50% |
| `object` | Non-walkable world props (trees, boulders, wheat, stumps, reeds, brambles, volcanic spires) | 20% |
| `background` | Seamless tiling textures (sand, grass, water, cobblestone, etc.) | 20% |
| `terrain` | Walkable hills/plateaus with wide flat tops for unit placement | 10% |

## Repository Map (What Is Where)

- `src/prompt_generator.py` — Builds prompt-data JSONL (captions + ComfyUI positive prompts)
- `src/dataset_generator.py` — Calls ComfyUI workflows, outputs raw images with metadata embedded in PNG tEXt chunks
- `src/png_metadata.py` — Pillow-based PNG `generation_metadata` tEXt chunk injection/extraction
- `src/prepare_dataset.py` — Post-curation: extracts captions, writes `.txt` files + `metadata.jsonl`
- `config/structure_workflow.json` — ComfyUI workflow for structure/object/terrain assets
- `config/background_tile_workflow.json` — ComfyUI workflow for background tiles
- `config/prompt_templates.json` — ComfyUI positive-prompt templates (not used for captions)
- `dataset/` — Working data root (prompts, raw dataset, hf dataset)
- `scripts/run_dataset.sh` — Example generator command

## Prerequisites

- A running ComfyUI endpoint reachable by `--comfy-url`
- Prompt data JSONL (generated by `src/prompt_generator.py`)
- Workflow API JSON files in `config/`

### Required LoRAs
Should be located under `ComfyUI/models/loras/` - see each LoRA repo for installation instructions.
- [lovis93/Flux-2-Multi-Angles-LoRA-v2](https://huggingface.co/lovis93/Flux-2-Multi-Angles-LoRA-v2)

### Required ComfyUI Nodes
Should be located under `ComfyUI/custom_nodes/` - see each node repo for installation instructions.
- [dimtoneff/ComfyUI-PixelArt-Detector](https://github.com/dimtoneff/ComfyUI-PixelArt-Detector)
- [Loewen-Hob/rembg-comfyui-node-better](https://github.com/Loewen-Hob/rembg-comfyui-node-better)

## End-to-End Flow

### 0) (Optional) Generate prompt data

```bash
python3 src/prompt_generator.py \
  --out-dir ./dataset/prompts \
  --seed 2026 \
  --total-target 28000 \
  --ratio-structure 0.50 \
  --ratio-object 0.20 \
  --ratio-background 0.20 \
  --ratio-terrain 0.10
```

Output: `dataset/prompts/generated_prompts.jsonl`

Each row: `id`, `asset_type`, `caption`, `positive_prompt`, `palette`

### 1) Generate images (raw)

```bash
./scripts/run_dataset.sh
```

Output:
- `dataset/raw/images/*.png` (with embedded `generation_metadata` tEXt chunk)
- `dataset/raw/dataset_manifest.jsonl`
- `dataset/raw/run_report.json`
- `dataset/raw/errors.jsonl`

### 2) Curate quality

Delete bad PNGs from `dataset/raw/images/` by visual review. `prepare_dataset.py` only includes assets whose PNGs still exist.

### 3) Prepare final dataset (HF)

```bash
python3 src/prepare_dataset.py ./dataset/raw/images --out-dir ./dataset/hf
```

Output:
- `dataset/hf/images/*.png` — curated PNGs
- `dataset/hf/images/*.txt` — caption text files (caption field only)
- `dataset/hf/metadata.jsonl` — HF `imagefolder`-compatible: `file_name`, `text`, `asset_type`, `palette`
- `dataset/hf/prepare_report.json`

### Caption Design

Captions are purely descriptive, template-free — no trigger tokens, no "pixel art 16x16", no tile-grid clauses. Trigger tokens are a fine-tuning concern injected later.

Example captions:
- Structure: *"a medieval watermill in a cozy harvest season setting, featuring careful repairs and patched materials, humble and practical character, medium detail, layered rooflines, oak timber and plaster, weathered, earthy palette"*
- Object: *"an ancient oak tree with storm-battered silhouette, layered silhouette, lush foliage and bark, healthy growth, earthy palette"*
- Terrain: *"a large hill with wide flat top, soft smooth side slopes, dense readable forms, packed earth and dry grass, weathered growth, earthy palette"*
- Background: *"sand texture with soft grain, subtle micro-dunes, gentle color noise, faint dithering, stable tile state, earthy palette"*

## Metadata Contract

### PNG embedded metadata (`generation_metadata` tEXt chunk)

The full prompt row is embedded: `id`, `asset_type`, `caption`, `positive_prompt`, `palette`, plus `image_path` added at generation time.

### HF metadata.jsonl

| Field | Description |
|---|---|
| `file_name` | Image filename (e.g., `mdv_0000001.png`) |
| `text` | Clean caption (no triggers, no style tokens) |
| `asset_type` | `background` / `structure` / `object` / `terrain` |
| `palette` | Palette name (e.g., `earthy palette`) |

## Reproducibility and Fine-Tuning Notes

- Use fixed prompt-data inputs and pinned workflow JSON files in `config/`.
- Keep generation controls (seed/guidance) in embedded PNG metadata.
- Curate by deleting PNGs only, then regenerate the HF dataset from remaining assets.
- Captions are intended for fine-tuning — trigger tokens are injected separately.
